# HexGaze 🔍

**HexGaze** — это минималистичный, обучающий, но при этом мощный и функциональный сетевой сниффер на языке C, написанный с нуля для операционных систем семейства Linux.

Проект создан без использования сторонних высокоуровневых библиотек захвата пакетов (таких как `libpcap`). Вся работа с сетью построена на фундаментальных системных вызовах Linux и сырых сокетах (**Raw Sockets**).

---

## 🛠 Технический стек и особенности

- **Язык**: C (стандарт C99/C11).
- **Платформа**: Linux.
- **Сетевое взаимодействие**: Использование сырых сокетов канального уровня (`AF_PACKET`, `SOCK_RAW`, `htons(ETH_P_ALL)`).
- **Режим работы карты**: Программный перевод сетевого адаптера в неразборчивый режим (**Promiscuous Mode**) через вызовы `ioctl`.
- **Безопасность**: Полный контроль границ буферов на каждом уровне разбора (L2/L3/L4), предотвращающий переполнение и выход за пределы выделенной памяти при получении некорректных или поврежденных пакетов.
- **Информативность**: Подробные комментарии в коде на русском языке, объясняющие принципы взаимодействия с ядром Linux и аппаратным обеспечением сетевого адаптера.

---

## 📐 Архитектура разбора пакета

HexGaze разбирает сетевой пакет слой за слоем (подобно стеку модели OSI):

1. **Layer 2 (Data Link)**: Разбирает заголовок **Ethernet II**. Определяет физические MAC-адреса источника и назначения, а также тип инкапсулированного протокола сетевого уровня (IPv4, ARP, IPv6).
2. **Layer 3 (Network)**: Разбирает заголовок **IPv4** (извлекает версию IP, длину заголовка с учетом опций, TTL, IP-адреса источника и получателя, тип инкапсулированного транспортного протокола).
3. **Layer 4 (Transport)**: Поддерживает детальный парсинг заголовков **TCP** (порты, Seq, Ack, управляющие флаги) и **UDP** (порты, длина датаграммы).
4. **Payload (Полезная нагрузка)**: Содержимое пакета после всех заголовков выводится в виде классического двухоконного **Hex Dump**:
   - Слева: относительное смещение.
   - По центру: байты в шестнадцатеричном виде.
   - Справа: символы в ASCII (печатные символы отображаются как есть, непечатные заменяются на точки `.`).

## 📦 Сборка проекта

Для компиляции проекта вам понадобятся компилятор `gcc` и утилита `make` (пакет `build-essential` в Debian/Ubuntu).

Выполните команду сборки в терминале:

```bash
make
```

Это создаст исполняемый файл `hexgaze` в текущей директории с оптимизацией `-O2` и включенными предупреждениями компилятора (`-Wall -Wextra`).

Для удаления бинарных файлов используйте:

```bash
make clean
```

## 🚀 Запуск сниффера

Сырые сокеты и программное переключение сетевой карты в Promiscuous Mode требуют прав суперпользователя (**root**).

### Способ 1. Запуск напрямую

Если вы не знаете точное имя интерфейса, запустите программу без аргументов:
```bash
sudo ./hexgaze
```
Программа выведет интерактивную таблицу всех доступных сетевых интерфейсов в системе, их адреса и подсказку по запуску.

Пример запуска на конкретном интерфейсе (например, `eth0` или `wlan0`):
```bash
sudo ./hexgaze eth0
```

### Способ 2. Запуск через Makefile

Вы можете использовать встроенную цель в `Makefile`, указав интерфейс в переменной `IFACE`:
```bash
make run IFACE=eth0
```

## 🧪 Инструкция по тестированию сниффера

Чтобы убедиться в работоспособности HexGaze, запустите сниффер в одном терминале, а во втором сгенерируйте сетевой трафик.

### Тест 1. Проверка ICMP (ping)

Запустите сниффер:
```bash
sudo ./hexgaze eth0
```

В другом терминале отправьте эхо-запросы (ICMP-пакеты) на любой адрес (например, DNS Google):
```bash
ping -c 3 8.8.8.8
```

*Что вы увидите в HexGaze:*
Программа перехватит входящие и исходящие пакеты ICMP, отобразит MAC и IP-адреса, укажет протокол L4 как `1 (ICMP)` и выведет Hex Dump тела ICMP-сообщения.

### Тест 2. Проверка TCP (curl / HTTP-запрос)

Поскольку современный веб-трафик шифруется по HTTPS, полезная нагрузка будет выглядеть как случайные байты. Чтобы наглядно увидеть текстовый Payload, можно выполнить запрос на тестовый незашифрованный HTTP-ресурс.

В другом терминале выполните:
```bash
curl http://neverssl.com
```

*Что вы увидите в HexGaze:*
Вы увидите целую сессию TCP-пакетов:
1. Пакеты с флагами `[ SYN ]` и `[ ACK SYN ]` — процесс трехэтапного рукопожатия (3-way handshake).
2. Пакет с флагом `[ ACK PSH ]` — содержащий HTTP-запрос. В Hex Dump справа вы четко прочитаете текстовые строки заголовков запроса:
   ```text
   GET / HTTP/1.1
   Host: neverssl.com
   User-Agent: curl/...
   Accept: */*
   ```
3. Входящие пакеты с ответом веб-сервера, где в Hex Dump будет виден HTML-код страницы.

### Тест 3. Проверка UDP (DNS-запрос)

Для генерации UDP-трафика можно выполнить DNS-запрос с помощью утилит `nslookup` или `dig`.

В другом терминале выполните:
```bash
nslookup google.com
```

*Что вы увидите в HexGaze:*
Сниффер захватит UDP-пакет на порт `53` (DNS). В выводе будет детально разобран UDP-заголовок, а в Hex Dump полезной нагрузки вы сможете увидеть байты DNS-запроса, содержащие строку `google.com`.
